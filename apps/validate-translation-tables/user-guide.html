Validate Translation Tables – User Guide

What it does: Two workflows — Validate (check/fix existing table) and Create (build new from Outcomes + myWSU). Requires Excel 365 or 2021+ for Validate export.

Validate workflow

1. Upload Outcomes source, Translation table, and myWSU. 2. Match Options → pick key columns (Translation input/output), validation mode (Key only or Key + name). 3. Name comparison (if used): pick Outcomes/myWSU name columns, threshold (default 0.8), ambiguity gap (0.03). 4. Select Columns, Keys, and Roles → pick included columns, key columns, optional role mapping (School, City, State, Country). Include City/State for high-confidence matching. 5. Validate Mappings → review cards → Download Full Report.

Excel review order

1. Review_Workbench (sortable/filterable) → main decision sheet. Only Decision is editable. Decisions: Keep As-Is, Use Suggestion, Allow One-to-Many, Ignore. 2. Final_Translation_Table → publish-ready. Not sortable — when finished, copy-paste values into a new sheet to sort/filter. 3. Translation_Key_Updates → changed keys only. 4. QA_Checks_Validate → pass before publish.

Create workflow

1. Upload Outcomes and myWSU. 2. Match method: key or name (key optional in name mode). 3. Select columns → include columns, optional roles. 4. Generate Translation Table → download Excel. See Review_Instructions_Create in workbook.

Tips

Review by exception. Check QA before publish. Use Suggestion: verify Suggested Key/School/City/State/Country first. Privacy: all processing is local in your browser.
